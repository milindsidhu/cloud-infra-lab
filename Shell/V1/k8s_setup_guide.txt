
Kubernetes Setup Guide for Master and Worker Nodes

This document provides a step-by-step guide for setting up Kubernetes on both the master and worker nodes. Some steps are common for both nodes, while others are specific to the master node.

---

Step 1: Load Kernel Modules on Both Nodes

On both the master and worker nodes, load the necessary kernel modules:

cat <<EOF | sudo tee /etc/modules-load.d/k8s.conf
overlay
br_netfilter
EOF

---

Step 2: Apply Kernel Modules on Both Nodes

Next, on both the master and worker nodes, run the following commands to load the modules:

sudo modprobe overlay
sudo modprobe br_netfilter

---

Step 3: Set Sysctl Parameters on Both Nodes

On both nodes, configure sysctl parameters for proper networking:

cat <<EOF | sudo tee /etc/sysctl.d/k8s.conf
net.bridge.bridge-nf-call-iptables  = 1
net.bridge.bridge-nf-call-ip6tables = 1
net.ipv4.ip_forward                 = 1
EOF

---

Step 4: Apply Sysctl Settings on Both Nodes

Apply the sysctl settings on both the master and worker nodes:

sudo sysctl --system

---

Step 5: Disable Swap on Both Nodes

Disable swap on both nodes to meet Kubernetes requirements:

sudo swapoff -a
(crontab -l 2>/dev/null; echo "@reboot /sbin/swapoff -a") | crontab - || true

---

Step 6: Define Kubernetes and CRI-O Versions

Define the versions of Kubernetes and CRI-O to use on both nodes:

KUBERNETES_VERSION=v1.32
CRIO_VERSION=v1.32

---

Step 7: Install Dependencies on Both Nodes

Install Dependencies for Adding Repositories:

Run the following commands to install dependencies on both nodes:

apt-get update
apt-get install -y software-properties-common curl

Add the Kubernetes Repository:

On both nodes, run the following commands to add the Kubernetes repository:

sudo mkdir -p /etc/apt/keyrings

# Add the Kubernetes signing key
curl -fsSL https://pkgs.k8s.io/core:/stable:/v1.32/deb/Release.key |     sudo gpg --dearmor | sudo tee /etc/apt/keyrings/kubernetes-apt-keyring.gpg > /dev/null

# Add the Kubernetes APT repo
echo "deb [signed-by=/etc/apt/keyrings/kubernetes-apt-keyring.gpg] https://pkgs.k8s.io/core:/stable:/v1.32/deb/ /" |     sudo tee /etc/apt/sources.list.d/kubernetes.list > /dev/null

Add CRI-O Repository:

On both nodes, run the following commands to add the CRI-O repository:

# Add CRI-O signing key
curl -fsSL https://download.opensuse.org/repositories/isv:/cri-o:/stable:/v1.32/deb/Release.key |     sudo gpg --dearmor | sudo tee /etc/apt/keyrings/cri-o-apt-keyring.gpg > /dev/null

# Add CRI-O APT repo
echo "deb [signed-by=/etc/apt/keyrings/cri-o-apt-keyring.gpg] https://download.opensuse.org/repositories/isv:/cri-o:/stable:/v1.32/deb/ /" |     sudo tee /etc/apt/sources.list.d/cri-o.list > /dev/null

Install Packages:

Now install the necessary packages for Kubernetes:

apt-get update
apt-get install -y cri-o runc kubelet kubeadm kubectl

Start CRI-O:

Start and enable CRI-O:

sudo systemctl enable crio
sudo systemctl start crio
crio --version
runc --version

---

Step 8: Install jq on Both Nodes

Install jq for parsing JSON:

sudo apt-get install -y jq

---

Step 9: Define Local IP Address on Both Nodes

Run the following commands on both nodes to define the local IP address for Kubernetes:

ifconfig

# Check for enp0s3 or similar and replace in
local_ip="$(ip --json a s | jq -r '.[] | if .ifname == "enp0s3" then .addr_info[] | if .family == "inet" then .local else empty end else empty end')"

echo "KUBELET_EXTRA_ARGS=--node-ip=$local_ip" | sudo tee /etc/default/kubelet > /dev/null

---

Step 10: Initialize the Master Node

On the master node only, run the following command to initialize Kubernetes with the defined IP address and other necessary settings:

echo $local_ip
# Replace the below IPADDR with the IP in $local_ip
export IPADDR="x.x.x.x"
export NODENAME=$(hostname -s)
export POD_CIDR="10.1.0.0/16"

sudo kubeadm init --apiserver-advertise-address=$IPADDR  --apiserver-cert-extra-sans=$IPADDR  --pod-network-cidr=$POD_CIDR --node-name $NODENAME --ignore-preflight-errors Swap --v=5

Get the Join Command:

If you skipped the join command during initialization, you can recreate it using the following:

sudo kubeadm token create --print-join-command

Configure kubeconfig:

On the master node, configure the Kubernetes kubeconfig file:

mkdir -p $HOME/.kube
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown $(id -u):$(id -g) $HOME/.kube/config

---

Step 11: Check Node Status on Master Node

Verify the node status on the master node:

kubectl get po -n kube-system

---

Step 12: Join the Worker Node

On the worker node, run the connection string obtained from the master node during the kubeadm init process:

sudo kubeadm join ......

---

Step 13: Install Calico Network Plugin for Pod Networking

On the master node, run the following command to install the Calico network plugin for Kubernetes:

kubectl apply -f https://raw.githubusercontent.com/projectcalico/calico/v3.29.3/manifests/calico.yaml

---

Step 14: Install Metrics Server

To install the Metrics Server on the master node, run the following commands:

Apply the Metrics Server Components:

kubectl apply -f https://github.com/kubernetes-sigs/metrics-server/releases/latest/download/components.yaml

Edit the Metrics Server Deployment:

kubectl -n kube-system edit deployment metrics-server

Add the following arguments under spec.containers.args:

- --kubelet-insecure-tls
- --kubelet-preferred-address-types=InternalIP,Hostname,InternalDNS,ExternalDNS,ExternalIP

---

Step 15: Verify Metrics Server and Node Metrics

Wait a few moments and then retry the command to get node metrics:

kubectl top node
